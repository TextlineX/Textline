# 项目状态记录

更新时间：2026-05-05

## 当前结论

这是一个正在推进中的全屏个人主页项目，当前主方向已经明确：

- 先把前端主页结构和视觉做稳
- 先用本地 mock 数据，不做真实 API
- 所有模块按板块和功能拆分
- 所有效果用 Less 分开管理
- Hero 作为首屏主视觉重点打磨
- About 作为 Hero 后的身份缓冲层，采用左侧叙事 + 右侧档案卡结构

## 当前进展

- Hero 的 Three.js 字符球已经从中间方块容器改成了整屏承载层。
- 球体尺寸已回收，不再使用放大的视觉比例。
- 滚动触发的球体运动已改为 2D 自由弹跳思路，不再是单轴动画回拉。
- 现在保留的是向下滚动触发脱落、边界反弹、空闲后自动回中心的状态机。
- 相关调试残留已清理，后续可以直接进入整理和提交阶段。

## 已确认的整体结构

前端页面按 `App.tsx` 组织为以下板块：

1. `Hero`
2. `About`
3. `Skills`
4. `Works`
5. `Experience`
6. `Playground`
7. `Contact`

## 已完成的基础能力

### 布局与导航

- `AppShell` 已负责整屏布局和板块切换
- `LogoMark` 已作为左上角入口，点击可回到首页顶部
- 右上角菜单与侧边栏已经拆成独立组件

### Hero 相关

- `NameBanner` 作为名字触发层
- `ThreeSphereScene` 作为 three.js 字符球主场景
- `GooeyReveal` 作为头像溢出层
- `MagneticCursor` 作为自定义磁吸光标
- `useMagneticCursor` 已把头像触发和普通吸附拆开
- 当前名字触发逻辑只允许由 `Textline` 触发头像，不允许其它磁吸目标误触发

### 背景与氛围

- Hero 背景已经朝更深、更收敛的 ASCII / 字符流方向调整
- 旧的视频主实现不再使用
- 背景只在 Hero 区域强调，后续板块可以切换到别的逻辑

### About 相关

- `About` 已从横向轨道收束为右侧叠放档案卡
- `About` 背景增加了光栅网格与随机发光点阵
- `About` 卡片已改为更实的纯色档案面板，不再是半透明玻璃卡
- `About` 顶部标题已收敛为单词 `IDENTITY`
- `About` 右侧卡片已整体左移，保留层叠感但降低右侧空洞
- `About` 磁吸只对当前激活卡片生效，避免整组一起抢鼠标

## 现在的约束

1. 头像只允许由名字触发，不允许其它磁吸目标触发。
2. Logo、菜单、侧边栏、板块卡片可以参与磁吸，但不触发头像。
3. 页面背景不能压住主元素底部。
4. 占位符必须极简，不能做成完整成品风格。
5. Hero 的视觉噪音要继续收敛，不能太满、太吵。

## 项目拆分状态

### 前端

- 技术栈：`Vite + React + TypeScript + Less`
- 当前负责视觉、交互、分板块展示
- 已开始使用独立的 `effects`、`hooks`、`data` 结构

### API

- 已预留独立 `api/` 目录
- 当前不做真实平台抓取
- 先用本地数据返回结果
- 后续如果要接 GitHub、掘金、B 站等平台，再单独扩展
