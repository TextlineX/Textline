# About 展示区资产交付规范

更新时间：2026-05-05

本文档用于约定 `About` 板块未来要接入的 3D 外壳、MP4 播放内容、贴图资源和交互热点。

目标不是做一个单纯的视频区，而是做一个可复用的展示模块：

- 外层是 3D / 伪 3D 的装置外形
- 中间是可替换的 MP4、图片或网页内容
- 上层保留 DOM 元素和磁吸光标
- 后续如果换主题，只替换资产，不重写交互结构

---

## 1. 总体目标

这个 `About` 区域建议作为主页里的“展示装置”使用，适合承载以下内容：

- 3D 外壳模型
- 屏幕区域视频
- DOM 文案、按钮、标签
- 磁吸光标
- 进入角度、透视、遮挡和轻交互

它的核心原则是：

1. 视觉可以复杂，但结构必须可替换。
2. 3D 外壳负责形态，屏幕内容负责信息，DOM 负责交互。
3. 光标吸附和内容展示分层，不互相耦合。

## 3. 3D 模型交付要求

### 3.1 推荐格式

- 优先：`.glb`
- 备选：`.gltf + 贴图文件`

### 3.2 推荐建模软件

- `Blender`
- 其它软件也可以，但最终最好导出为 `glb`

### 3.3 模型结构建议

模型最好至少拆成下面几个逻辑部件：

- `Root`
- `Body`
- `ScreenFrame`
- `Screen`
- `Stand`
- `BackShell`
- `Pivot`
- `ScreenMask`
- `CursorAnchor`

其中：

- `Body` 表示主体壳体
- `ScreenFrame` 表示屏幕边框
- `Screen` 表示屏幕平面
- `ScreenMask` 表示屏幕裁切或显示区域
- `Pivot` 表示整体旋转中心
- `CursorAnchor` 表示鼠标磁吸参考点

### 3.4 模型原点和朝向

请尽量遵守以下约定：

- 模型原点尽量放在主体中心
- 模型正面朝向要固定
- 推荐正面朝 `+Z` 或 `-Z`
- 不要导出后模型一进来就横着、倒着、斜着
- 如果有明显的屏幕正面，请保证屏幕面朝外

### 3.5 模型尺寸

建议先按以下思路做：

- 主体尽量控制在一个装置级别，不要过大
- 如果是播放器/终端外壳，优先按 `16:9` 屏幕比例思考
- 屏幕区域最好边界清晰
- 外壳不要遮住屏幕可视区

建议参考尺度：

- 宽屏面板：`16:9`
- 方形面板：`1:1`
- 竖屏面板：`9:16`

如果你没法定尺寸，可以先按 `16:9` 做，后面我再适配。

### 3.6 面数和复杂度

建议先保持轻量：

- 尽量别做过碎的高面数模型
- 能合并的部件尽量合并
- 先保证轮廓、结构和屏幕关系清楚
- 如果有高模和低模，优先提供低模版本

建议目标：

- 单个 `glb` 最好先控制在 `20MB` 以内
- 更理想是 `5MB ~ 10MB`

### 3.7 材质要求

建议材质保持清晰，不要太复杂：

- 主体可以是金属、塑料、磨砂、玻璃任一种
- 但请在 `notes.txt` 里写清楚材质意图
- 如果屏幕区域需要自发光，请单独标记
- 如果有透明、半透明、发光边缘，请明确说明

### 3.8 贴图命名建议

如果你提供贴图，建议使用下面的命名方式：

- `body_albedo.png`
- `body_normal.png`
- `body_roughness.png`
- `body_metalness.png`
- `body_ao.png`
- `screen_mask.png`
- `screen_emissive.png`
- `screen_frame.png`

### 3.9 建模时最好额外给出的内容

如果方便，请顺带给：

- 模型正面截图
- 45 度截图
- 侧面截图
- Blender 场景截图

这样我能更快对齐视角和屏幕区域。

## 4. MP4 交付要求

### 4.1 推荐格式

- 格式：`.mp4`
- 编码：`H.264`
- 音频：如果不需要，建议直接静音

### 4.2 推荐分辨率

按使用场景选：

- 横屏展示：`1920x1080`
- 竖屏展示：`1080x1920`
- 如果未来要上更高清，也可以给 `2560x1440`

### 4.3 推荐帧率

- 默认：`30fps`
- 如果画面运动较多，可以给 `60fps`

### 4.4 视频内容建议

MP4 最好承担以下角色之一：

- 屏幕内内容
- 演示内容
- 视觉背景
- 装置状态动画

建议注意：

- 不要放播放器控件
- 不要有黑边
- 不要有水印
- 不要把关键内容贴到边缘
- 如果要循环，首尾尽量能无缝衔接

### 4.5 MP4 文件大小

建议先控制在：

- `20MB ~ 50MB` 以内更稳
- 如果只是短循环，越小越好

### 4.6 MP4 命名建议

建议命名：

- `about-showcase.mp4`
- `about-showcase-loop.mp4`
- `about-showcase-intro.mp4`

如果有封面帧，建议单独给：

- `about-showcase-poster.webp`

## 5. 网页 DOM 与磁吸光标约定

这个展示区不是纯视频播放区，它还会叠一层网页元素。

### 5.1 可叠加的 DOM 元素

可以放：

- 标题
- 副标题
- 按钮
- 标签
- 链接
- 简短说明
- 状态指示

### 5.2 磁吸约定

磁吸光标会继续沿用项目里的通用逻辑，所以建议：

- 交互目标加 `.magnetic-target`
- 需要更强磁吸的元素可以加 `data-magnetic-shell`
- 需要额外触发行为的元素可以加 `data-avatar-trigger`

### 5.3 交互区域建议

如果屏幕上有可点击或可悬停区域，建议命名：

- `Hotspot-Play`
- `Hotspot-Title`
- `Hotspot-Action`
- `Hotspot-Nav`

这样后面我可以直接把这些区域接到磁吸和事件里。

## 6. 视觉结构建议

为了让这个区域更像一个“装置”，建议结构拆成三层：

### 6.1 底层

- 3D 外壳
- 环境背景
- 光照

### 6.2 中层

- MP4 或视频纹理
- 屏幕内 DOM
- 边框发光

### 6.3 顶层

- 磁吸光标
- 悬停状态
- 文字高亮

## 7. 如果你用 Blender 制作，建议这样导出

### 7.1 导出前检查

- 模型原点是否正确
- 轴向是否统一
- 屏幕是否单独命名
- 材质是否能正确预览
- 非必要细节是否已删除

### 7.2 导出建议

- 优先导出 `.glb`
- 如果有贴图，确认一起打包
- 如果有多材质，先统一命名
- 如果有空物体，请保留命名

### 7.3 导出后检查

- 打开后模型是否正向
- 屏幕是否在正确位置
- 材质是否丢失
- 透明区域是否正常
- 是否有异常缩放
