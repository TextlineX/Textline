# 视频背景分段方案

## 目标

为 `Textline` 首页引入一条预烘焙视频背景主轴，用低性能开销承载高完成度的视觉叙事。

当前页面不是原生长滚动，而是由 `AppShell` 通过 `activeIndex` 做整屏切换，因此背景不应该依赖滚动进度驱动，而应该依赖当前板块状态驱动。

## 总体原则

1. 视频负责氛围，不负责主交互。
2. 前景负责叙事主体，包括名字、three.js 球、鼠标准星、gooey。
3. 背景视频只做“可切换的连续长轴”，不做重逻辑。
4. 每个板块对应一个视频段落和一个蒙版预设。
5. 中心主视觉区域必须避让，不让背景直接穿过 `Textline` 和球体。

## 当前板块结构

当前 `src/App.tsx` 里实际存在 7 个板块：

1. `hero`
2. `about`
3. `skills`
4. `works`
5. `experience`
6. `playground`
7. `contact`

对应导航数据在 [`src/data/siteData.ts`](../src/data/siteData.ts)。

## 板块 -> 视频段落 -> 蒙版映射

| 板块 | 视觉段落 | 视频内容建议 | 蒙版形态 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| `hero` | 叙事起点 | Text Ribbon Sphere 缓慢自转 | 中心圆孔蒙版 | 给名字和球体留出中心主视觉 |
| `about` | 自我说明 | 球体延展为更松散的字符雾 | 椭圆扩散蒙版 | 作为 hero 的叙事延展，不必太跳 |
| `skills` | 能力结构 | 平行数据流、扫描线、模块化片段 | 水平扫描线蒙版 | 体现工程化和分层感 |
| `works` | 作品陈列 | 字符线缆形成网格或展示墙 | 矩形视窗蒙版 | 更像作品柜和卡片背板 |
| `experience` | 时间线 | 横贯屏幕的连续脉冲线 | 线性路径蒙版 | 适合时间顺序叙事 |
| `playground` | 实验区 | 流体碰撞、glitch、字符扰动 | 不规则碎裂蒙版 | 允许更混沌、更实验 |
| `contact` | 收束结尾 | 字符向中心坍缩成奇点 | 全屏渐隐蒙版 | 用于收尾和离开页面 |

## 颜色方向

背景视频也应该服从 `Abyssal Ember` 主题，不要回到普通霓虹风。

建议色阶：

1. 基础背景：`#050505`
2. 主文字：`#E0E0E0`
3. 主强调：`#FF2E2E`
4. 辅助强调：`#00FFFF`
5. 次强调：`#8A2BE2`
6. 低亮线条：`rgba(224, 224, 224, 0.22)`

## 前端落地方式

### 1. 背景层

当前实现已经收敛为单层 ASCII 背景流，旧的 `VideoBackdrop` 方案属于历史方案，不再作为主实现。

当前背景层职责：

- 渲染单层字符流
- 接收当前板块 id 或全局状态
- 根据板块状态切换蒙版预设

### 2. 板块状态表

建议在数据层维护一份结构化配置：

```ts
type BackgroundSection = {
  id: string
  segment: number
  maskPreset: 'circle' | 'scan' | 'window' | 'path' | 'glitch' | 'fade'
  intensity: number
}
```

页面切换到哪个板块，就把当前状态同步给背景层。

### 3. 时间轴

如果视频是单条长轴，建议把它切成 7 段：

- `0% - 15%`: hero
- `15% - 28%`: about
- `28% - 42%`: skills
- `42% - 58%`: works
- `58% - 72%`: experience
- `72% - 88%`: playground
- `88% - 100%`: contact

如果后期想更自由，也可以让 `activeIndex` 映射到 `video.currentTime` 的区间，而不是单纯切帧。

## 蒙版策略

### Hero

- 中心使用大圆孔
- 给 `Textline` 和 three.js 球留白
- 允许鼠标驱动的小幅变化

### Skills

- 蒙版拉成水平扫描线
- 视觉上像数据扫描和能力读取

### Works

- 蒙版更像展示窗
- 适合卡片、缩略图、项目入口

### Experience

- 蒙版沿时间轴延展
- 重点是“连续感”

### Playground

- 蒙版破碎、不规则
- 允许更强的 glitch 感

### Contact

- 背景收束
- 蒙版渐隐到黑场

## 关于视频格式

建议优先考虑：

1. `webm`
2. `mp4`

如果想做透明背景，优先考虑 `webm`；如果最终只是黑底循环，`mp4` 更容易兼容。

## 关于 ASCII 化

ASCII 风格可以作为视频内容的一种预处理方向，但不建议把所有板块都做成 ASCII。

更合理的做法是：

- `hero` 用高密度 ASCII / 字符线球
- `skills` 用低密度 ASCII 数据流
- `playground` 用 glitch ASCII
- 其他板块偏克制，避免整站都像实验 demo

## 结论

这条背景视频不是单纯的装饰，而是页面叙事的一部分。

最合适的实现方式是：

1. 一个循环视频主轴
2. 七个板块状态
3. 七种蒙版预设
4. 一个前景叠层体系

这样后续你无论是换视频、换段落，还是换蒙版，都不会破坏主体结构。
