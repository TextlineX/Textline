# 超级个性化个人主页结构设计

版本：`v0.1`
状态：结构提案

## 1. 项目定位

这个站点不是普通的个人介绍页，而是一个“可展示能力的个人主页作品”。

核心目标：

- 让访客第一眼感受到你的风格
- 让页面本身成为作品，而不是仅仅承载作品
- 允许后续持续叠加 `three.js`、`matter.js`、`anime.js` 等动态效果
- 兼顾展示、交互、传播和可扩展性

## 2. 设计原则

- 先建立结构，再填充视觉和特效
- 所有效果都服务于“个人表达”，不做无意义堆叠
- 页面应分层组织，避免动效和内容互相干扰
- 主页要支持从“静态浏览”到“强交互体验”的渐进增强

## 3. 总体架构

建议将整站拆成三层：

### 3.1 舞台层

负责视觉氛围和沉浸感：

- 全屏背景
- 粒子、光效、流体、网格、噪点、扫描线等视觉元素
- 可接入 `three.js` 或 Canvas/WebGL

### 3.2 内容层

负责信息表达：

- 个人身份
- 技能和技术栈
- 作品展示
- 时间线 / 经历
- 联系方式

### 3.3 交互层

负责增强记忆点：

- 鼠标跟随效果
- 卡片拖拽、碰撞、弹性动画
- 悬浮状态联动
- 页面滚动驱动动效

## 4. 页面信息架构

建议采用“单页主站 + 锚点分区”的结构。这样更适合个人主页，也方便后面做滚动联动效果。

### 4.1 导航层

顶部或侧边固定导航，包含：

- Home
- About
- Skills
- Works
- Experience
- Playground
- Contact

导航的作用不是复杂跳转，而是快速锚定各个视觉区块。

### 4.2 首屏 Hero

首屏是最重要的一屏，建议作为“视觉主场”。

包含内容：

- 姓名 / 昵称
- 一句强记忆点的个人宣言
- 当前身份标签，例如“前端 / 视觉交互 / 创作型开发者”
- 主按钮：查看作品、联系我
- 次按钮：进入交互区

首屏建议叠加：

- 背景动效
- 光晕或粒子
- 轻微视差
- 鼠标跟随反馈

### 4.3 关于我 About

这里不是长篇自我介绍，而是“快速建立印象”。

建议内容：

- 你是谁
- 你擅长什么
- 你希望给别人留下什么印象
- 你的创作方向

可以用卡片、时间轴、标签、数据块的方式呈现。

### 4.4 技能与能力 Skills

展示技术能力，但不要做成流水账。

建议拆成三类：

- 前端能力
- 动效与视觉能力
- 工程化与工具链能力

展示形式建议：

- 雷达图
- 技能墙
- 技术徽章
- 动态计量条

### 4.5 作品展示 Works

这是主页的“证据区”。

每个作品建议包含：

- 项目名称
- 一句话说明
- 使用技术
- 关键亮点
- 预览入口 / GitHub / Demo

推荐展示方式：

- 重点作品大卡片
- 辅助作品小卡片
- 横向滑动作品带
- 悬浮展开详情

### 4.6 经历时间线 Experience

用于表达你的成长轨迹。

可以放：

- 学习路径
- 工作经历
- 重要项目节点
- 技术转变节点

建议使用时间轴样式，而不是传统简历样式。

### 4.7 互动实验区 Playground

这是个人主页的差异化重点。

可以放一些“会动的东西”：

- `three.js` 场景
- `matter.js` 物理交互
- `anime.js` 编排动画
- 图形实验
- 视觉测试组件

这个区域的目标不是信息密度，而是展现你的创造力和技术表达。

### 4.8 联系区 Contact

最后一屏负责转化。

建议包含：

- 邮箱
- GitHub
- 社交链接
- 可下载简历
- 一句结束语

## 5. 推荐页面结构

建议的 DOM 结构如下：

```text
App
├── AppShell
│   ├── BackgroundStage
│   ├── CursorEffects
│   ├── TopNav
│   ├── MainContent
│   │   ├── HeroSection
│   │   ├── AboutSection
│   │   ├── SkillsSection
│   │   ├── WorksSection
│   │   ├── ExperienceSection
│   │   ├── PlaygroundSection
│   │   └── ContactSection
│   └── Footer
└── GlobalOverlays
```

## 6. 组件拆分建议

### 6.1 基础布局组件

- `AppShell`
- `Section`
- `Container`
- `Grid`
- `Stack`
- `GlassCard`

### 6.2 业务展示组件

- `HeroBanner`
- `ProfileBadge`
- `SkillMatrix`
- `ProjectCard`
- `TimelineItem`
- `ContactLinks`

### 6.3 动效组件

- `ParticleField`
- `MouseFollower`
- `FloatingOrbs`
- `ScrollReveal`
- `AnimatedCounter`
- `PhysicsSandbox`

### 6.4 数据驱动组件

这些组件尽量不要写死内容，建议由配置驱动：

- `navItems`
- `skills`
- `projects`
- `experience`
- `socialLinks`

## 7. 推荐目录结构

建议后续演进为下面这种结构：

```text
src/
├── assets/
├── components/
├── sections/
├── effects/
├── data/
├── hooks/
├── styles/
├── utils/
├── App.tsx
└── main.tsx
```

### 7.1 `components`

通用组件库，不绑定具体页面语义。

### 7.2 `sections`

页面分区组件，直接对应首页模块。

### 7.3 `effects`

专门放视觉和动效能力，避免和业务组件耦合。

### 7.4 `data`

存放个人信息、作品、技能等配置。

### 7.5 `styles`

存放全局样式、主题变量、字体定义、动画变量。

## 8. 技术分层建议

### 8.1 基础层

- `Vite`
- `TypeScript`
- `React`

### 8.2 样式层

- `Tailwind` 用于快速搭建布局和通用样式
- `Less` 用于复杂视觉区、定制主题和局部结构样式

### 8.3 动效层

- `anime.js` 负责常规 DOM 动画
- `matter.js` 负责物理交互
- `three.js` 负责 3D 场景和沉浸式视觉

### 8.4 数据层

- 主页内容配置化
- 作品、技能、链接都尽量抽离到数据文件

## 9. 体验目标

这个主页最终应该满足以下体验：

- 第一眼有记忆点
- 第二眼能看出你会什么
- 第三眼能感受到你有表达欲和技术审美
- 继续浏览后能找到你的作品和联系入口

## 10. 下一阶段建议

如果你认可这个结构，下一步建议按这个顺序推进：

1. 先搭页面骨架，不上复杂特效
2. 再把首屏 Hero 做成强视觉区
3. 然后补 About / Skills / Works / Contact
4. 最后接入 `three.js`、`matter.js`、`anime.js` 作为实验区

## 11. 当前结论

这个网站应该是：

- 单页为主
- 视觉驱动
- 内容模块化
- 特效可插拔
- 适合持续迭代的个人作品主页
